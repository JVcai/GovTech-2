GovTech: Merit-Based Subsidy Allocation System (v3.0)

Интеллектуальная система меритократического распределения сельскохозяйственных субсидий для Министерства сельского хозяйства РК.

Кейс №2: Переход от принципа «живой очереди» к распределению на основе эффективности (ROI), социального вклада и потенциала роста.

Проблема и бизнес-цели

Проблема (As-Is):

Принцип FIFO: Субсидии уходят тем, кто первым подал заявку, независимо от эффективности хозяйства.

Неэффективное освоение: Бюджет может закончиться на низкоэффективных проектах, в то время как стратегически важные хозяйства остаются без поддержки.

Секторальный дисбаланс: Животноводство и садоводство с длинными циклами проигрывают «быстрым» отраслям (птицеводство).

Решение (To-Be):

Внедрение системы AI-Driven Merit Scoring, которая ранжирует заявителей по интегральному показателю эффективности, обеспечивая максимальный возврат на каждый инвестированный тенге государственного бюджета.

🛠 Архитектура и Технологический стек

Проект реализован как гибридная аналитическая платформа (Machine Learning + Operations Research):

Язык: Python 3.10+

ML Engine: XGBRegressor (Градиентный бустинг над решающими деревьями).

Explainability: SHAP (Shapley Additive Explanations) для обоснования решений.

Optimization: Алгоритм Waterfall Allocation для управления бюджетом.

API: FastAPI для интеграции с государственными базами данных (ISS).

🧠 Описание алгоритмов

1. Интегральный скоринг (Performance Score)

Вместо классификации «Да/Нет», модель предсказывает Performance Score (0–100).
Ключевые факторы (Features):

Yield Index: Эффективность выхода продукции на единицу запрошенных средств.

Social Impact: Количество создаваемых рабочих мест на 1 млн KZT субсидии.

Sector Rank: Относительная позиция хозяйства внутри своего сектора (нормализация по 9 направлениям).

History Factor: Дисциплина освоения прошлых траншей (на основе исторических данных ISS).

2. Секторальная нормализация

Для обеспечения справедливости система сравнивает «яблоки с яблоками». Пчеловод не конкурирует с агрохолдингом КРС. Каждая заявка получает Sector Rank Percentile, который составляет 30% итогового веса.

3. Waterfall Allocation (Алгоритм «Водопада»)

Compliance: Жесткая фильтрация (налоговая задолженность, реестр недобросовестных поставщиков).

Ranking: Сортировка по убыванию Performance Score.

Cap Policy: Ограничение максимальной выплаты (20% от бюджета) для предотвращения монополизации.

Distribution: Линейное распределение до исчерпания лимита. Поддержка Partial Funding (частичного финансирования) для заявок на границе остатка бюджета.

📂 Структура репозитория

app.py — Ядро системы: предобработка данных, обучение регрессионной модели и API-слой.

test_distribution.py — Модуль симуляции: загружает реальный реестр ISS (36k+ строк), проводит ранкинг и формирует финальный отчет.

requirements.txt — Список зависимостей для развертывания.

data/ — Директория для входящих данных (Excel/CSV).


БЫСТРЫЙ СТАРТ:
Установка:
pip install -r requirements.txt

Запуск симуляции (Batch Processing):
Программа проанализирует реестр, обучит модель и выведет шорт-лист выплат:

python test_distribution.py --budget 5000000--cap 0.15


РЕЗУЛЬТАТЫ И KPI

Освоение бюджета: 16%.

Масштабируемость: Обработка >36,000 заявок за 90 секунд.

Слабая сторона: Слишком жесткая система оценивания и в итоге маленькое освоение бюджета

Прозрачность: Каждая позиция в рейтинге снабжена текстовым обоснованием факторов роста.
